Storing data in R

So far we have stored single numbers using <-, for example:

x <- 5

To store a sequence of numbers, combine them using the c() function.
A sequence of elements of the same type is called a vector.

For example:

myfirstvector <- c(1, 5, 3, 7)
myfirstvector

R prints:

[1] 1 5 3 7

You can carry out arithmetic on every element of the vector:

myfirstvector + 5

Returns:

[1] 6 10 8 12

Vectors can also contain words (‚Äúcharacter‚Äù data):

wordsvector <- c("cat", "dog", "parrot", "peppapig")
wordsvector

R prints:

[1] "cat" "dog" "parrot" "peppapig"

You may use single or double quotation marks; both work.

You can check the type of an object using class():

class(wordsvector)

Because addition only makes sense for numbers, R will give an error if you try an invalid operation:

wordsvector + 5

Finally, if you mix types inside c(), R converts everything to the most general type (usually characters):

mysecondvector <- c(4, "cat")
mysecondvector

Returns:

[1] "4" "cat"

Putting all of this together, you now know how R stores individual values, sequences of numbers, and sequences of text, and how R decides what type each vector should be. These tools form the foundation for reading, inspecting, and describing real datasets ‚Äî skills you will now begin applying in your coursework.

Convert selected variables to factors

What does factor() do?

factor() is a function in R that turns a variable into a factor, which is R‚Äôs way of storing categorical data (groups, labels, categories).

Many datasets import text categories as "character" data.
Using factor() tells R that these values represent categories, not free-text.

movies$Genre <- factor(movies$Genre)
movies$LeadStudio <- factor(movies$LeadStudio)
movies$Year <- factor(movies$Year)

For example:

  • movies$Genre contains labels such as "Comedy", "Drama", "Horror".
  • factor(movies$Genre) converts these labels into a categorical variable.

We use:

movies$Genre <- factor(movies$Genre)

to replace the original column with a factor version of itself.

The $ operator

The $ symbol is used to select a single column from a dataset.
For example,
movies$Genre means ‚Äúthe Genre column inside the movies dataset‚Äù.

Check the updated structure:

After converting character variables into factors, the output of summary() becomes much more informative for categorical data.

Instead of only showing "Length", "Class", and "Mode", R now displays the number of observations in each category (for example, how many films belong to each genre or studio).

This is why converting categorical variables to factors is useful: it allows R to treat them properly as categories and summarise them in a meaningful way.

How to read the summary output

How to read the summary output

The summary() function gives a quick statistical overview of every variable in the dataset. For numeric variables, it shows:

  • Min. ‚Äì the smallest value

  • 1st Qu. ‚Äì the value below which 25% of the data lie

  • Median ‚Äì the middle value

  • Mean ‚Äì the average

  • 3rd Qu. ‚Äì the value below which 75% of the data lie

  • Max. ‚Äì the largest value

  • NA‚Äôs ‚Äì how many values are missing

These numbers help you understand the overall range and spread of each variable. For example,

 Budget ranges from 0.9 to 365 million dollars, and

 RottenTomatoes scores run from 0 to 99.

For categorical variables‚Äînow stored as factors‚Äîsuch as Genre or LeadStudio, the summary shows the number of observations in each category. This lets us quickly see which genres or studios appear most often in the dataset.

It is also helpful to check whether the minimum or maximum values look realistic. For instance, the Profitability variable has a maximum of 10176%, meaning a film supposedly earned more than one hundred times its budget. This is unusually high and suggests there may be some data entry errors that need to be investigated later.

Overall, summary() provides a useful first look at the structure of the dataset and can help identify potential issues such as unrealistic values or many missing entries.

Detecting unusual values

Sometimes it is helpful to look for values that seem unrealistic.
From the summary output, we noticed that Profitability has a maximum value of 10,176%, which suggests a film supposedly earned more than one hundred times its budget. This is extremely unlikely and probably indicates a data entry error.

To inspect these cases more closely, we can filter the rows where profitability is greater than 1000%:

a > b tests whether a is greater than b. a < b tests whether a is smaller than b. a == b tests whether a is equal to b; notice the double equal sign! You can also use >= or <=
